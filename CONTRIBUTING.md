# Contributing to Teleop Platform

Thank you for your interest in contributing to the Teleop Platform! We welcome contributions from the community to help improve and expand this project.

## Contributor License Agreement (CLA)

In order to clarify the intellectual property license granted with Contributions from any person or entity, the Teleop Platform project requires that each Contributor sign a Contributor License Agreement ("CLA").

By signing the CLA, you verify that you are the author of the contribution or otherwise have the right to submit it, and you grant the project maintainers a license to use and relicense your contributions (including for commercial purposes).

**You must sign the CLA before your Pull Request can be merged.**

### How to Sign
1.  Download the CLA form [Link to CLA Form - Placeholder].
2.  Sign the document (digital signature is accepted).
3.  Email the signed copy to `cla@teleop-platform.com` (or upload it to the designated CLA portal).

*Note: If you are contributing on behalf of your employer, please ensure you have permission to sign the CLA or have your employer sign the Corporate CLA.*

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

## How to Contribute

1.  **Fork the repository** on GitHub.
2.  **Clone the project** to your own machine.
3.  **Create a new branch** for your feature or bug fix.
    ```bash
    git checkout -b feature/my-new-feature
    ```
4.  **Make your changes** and commit them.
    ```bash
    git commit -m "feat: Add some amazing feature"
    ```
5.  **Push your branch** to your fork.
    ```bash
    git push origin feature/my-new-feature
    ```
6.  **Open a Pull Request** (PR) against the `main` branch.

## Coding Guidelines

*   Follow existing code style (Python PEP 8).
*   Add comments and documentation for new features.
*   Ensure all tests pass before submitting.
*   Update `TODO-Notion.md` if your changes affect the roadmap.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on the GitHub repository. Provide as much detail as possible to help us understand and resolve the issue.

Thank you for your contribution!
